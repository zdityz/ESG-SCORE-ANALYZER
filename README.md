# ESG Scorer AI

An AI-powered Environmental, Social & Governance (ESG) scoring engine built for Indian companies. Upload sustainability reports or fill in data manually to get scored, benchmarked against industry peers, and receive actionable recommendations — all via a REST API.

---

## Features

- **AI Benchmarking** — A Gradient Boosting model trained on 5,000 synthetic Indian companies predicts expected energy usage based on revenue, headcount, and sector, so your score is always judged against realistic peers, not a one-size-fits-all threshold.
- **Dynamic Industry Weighting** — E/S/G weights shift automatically based on sector risk (e.g., Cement/Steel is 60% Environmental; IT/Services is 50% Social).
- **Document Parsing** — Upload `.xlsx` or `.pdf` sustainability reports; the engine extracts key metrics automatically.
- **PDF Report Generation** — Get a formatted audit report with a radar chart and AI recommendations in one API call.
- **Actionable Recommendations** — Specific alerts for energy overuse, safety incidents, regulatory fines, and more.

## Quickstart

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Train the AI benchmark model

This only needs to be run once. It generates `esg_benchmark_model.pkl` in the project root.

```bash
python train_model.py
```

### 3. Start the API server

```bash
uvicorn final:app --reload
```

The API will be available at `http://localhost:8000`. Interactive docs at `http://localhost:8000/docs`.

## API Reference

### `POST /calculate_score`

Calculates ESG scores from structured JSON input.

**Request body:**

```json
{
  "company_profile": {
    "industry": "IT/Services",
    "annual_revenue_inr": 50000000,
    "total_employees": 200
  },
  "environmental": {
    "total_energy_consumption_kwh": 300000,
    "renewable_energy_kwh": 90000,
    "total_water_consumption_liters": 10000,
    "waste_generated_kg": 5000,
    "waste_recycled_kg": 3500
  },
  "social": {
    "female_employees": 80,
    "employees_with_disabilities": 5,
    "safety_accidents_count": 0,
    "employees_trained_count": 180,
    "complaints_received_sexual_harassment": 0
  },
  "governance": {
    "has_sustainability_committee": true,
    "regulatory_fines_paid_inr": 0,
    "policies_implemented": ["Anti-Bribery", "Whistleblower", "Data Privacy"]
  }
}
```

**Response:**

```json
{
  "scores": {
    "environmental": 78.5,
    "social": 91.0,
    "governance": 95.0,
    "total": 87.3
  },
  "recommendations": []
}
```

---

### `POST /generate_graph`

Same request body as `/calculate_score`. Returns a PNG radar chart of the three pillar scores.

---

### `POST /generate_report`

Same request body as `/calculate_score`. Returns a downloadable PDF audit report containing the executive summary, radar chart, scores, and AI recommendations.

---

### `POST /extract_from_file`

Upload one or more `.xlsx` or `.pdf` sustainability reports. Returns a pre-filled JSON structure ready to pass to `/calculate_score`.

```bash
curl -X POST http://localhost:8000/extract_from_file \
  -F "files=@sustainability_report.pdf"
```

## Scoring Methodology

Each pillar is scored out of 100 and then combined using industry-adjusted weights.

### Environmental (default weight: 40%)

| Component | Max Points | Method |
|-----------|-----------|--------|
| Energy efficiency vs AI benchmark | 50 | AI prediction vs actual usage |
| Renewable energy percentage | 30 | Linear scale up to 30% renewable |
| Waste recycling rate | 20 | Recycled / Total generated |

### Social (default weight: 30%)

| Component | Max Points | Method |
|-----------|-----------|--------|
| Gender diversity | 40 | Linear scale up to 40% female workforce |
| Safety record | 30 | 0 accidents = 30 pts; −10 per accident |
| Employee training coverage | 30 | Linear scale up to 50% trained |

### Governance (default weight: 30%)

| Component | Max Points | Method |
|-----------|-----------|--------|
| Policies implemented | 50 | 15 pts per policy, capped at 50 |
| Regulatory fines | 30 | 30 pts if zero fines |
| Sustainability committee | 20 | Yes/No |

### Industry Weight Adjustments

| Industry | E | S | G |
|----------|---|---|---|
| IT/Services | 20% | 50% | 30% |
| Cement/Steel | 60% | 20% | 20% |
| Pharma | 60% | 20% | 20% |
| Manufacturing / Retail | 40% | 30% | 30% |

## AI Benchmark Model

`train_model.py` generates a **Gradient Boosting Regressor** trained on 5,000 synthetic Indian companies.

**Input features:** Annual revenue (INR), total employees, industry type  
**Output:** Predicted energy consumption (kWh) — the "expected" baseline for a company of that size and sector

The training data intentionally includes outliers:
- **10% "Bad Apples"** — old machinery, 40–80% over expected usage
- **5% "Green Leaders"** — solar/LED optimised, 20–40% under expected usage
- **85% Standard** — normal ±10% variation

This teaches the model to benchmark against realistic industry norms, not best-case scenarios, making the gap meaningful when it flags a company as inefficient.
